An authoritative bridge between Search Console signals and the universal AI Agent ecosystem.

The Experience

GSC MCP Server Connected
"Is my content being cannibalized by AI Overviews? Run an audit."
gsc-search.get_search_appearance_audit() Searching for High Position / Low CTR patterns...
I've identified 34 queries with high cannibalization risk. Your page on "system prompts" ranks #1.1 but has a 0.4% CTR. Users are likely getting the full answer from the Google snippet.
"Check the technical health of that page. Why is CTR so low?"
gsc-search.get_technical_citation_audit() Inspecting URL: https://yoursite.com/system-prompts
The page is technically perfect (Indexed, Mobile-Friendly). The low CTR isn't a technical error—it's Intent Satisfaction. You should add a dynamic tool or interactive element to pull users in.

Core Benefits

  • Stop Cannibalization: Automatically detect where you are serving as a "Silent Reference" in AI Overviews without earning clicks.
  • Understand Intent: Segment traffic into "Searchers" vs "Prompters" to see how audience behavior is shifting.
  • Technical Verdicts: Get real-time indexing and usability status for your most visible pages directly in the chat.
  • Context Efficiency: Analyzes large properties without overflowing your Agent's context window through server-side aggregation.

Quick Setup

  • Install the package 
    pip install google-search-console-mcp
  • Connect Google Search Console

    Create a Service Account JSON key in Google Cloud and grant it access to your GSC property.

  • Configure your AI Agent

    Add the server to your agent configuration file.

    {
  "command": "gsc-mcp",
  "env": {
    "GSC_SITE_URL": "sc-domain:yoursite.com",
    "GOOGLE_APPLICATION_CREDENTIALS": "/path/to/key.json"
  }
}
